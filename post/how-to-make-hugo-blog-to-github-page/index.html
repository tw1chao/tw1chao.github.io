前言#

將過往在 HackMD 整理好的資料，轉移到自己的 Github Page 上，一開始在挑選建立 Blog 的工具上找到了 Hexo ，後來聽聞 Hugo 編譯（產生靜態頁面）的速度比 Hexo 快了很多，果斷捨棄（停損） Hexo 建置 Blog 的方法。 以下將紀錄在 Linux 上使用 brew 安裝 hugo 並建立 Blog 的方法。

教學開始#

這邊安裝的 Hugo 版本為 hugo v0.96.0+extended linux/amd64 BuildDate=unknown

作業系統環境 Linux 120-117-81-57 5.9.13 #1 SMP Tue Dec 15 10:25:24 CST 2020 x86_64 GNU/Linux

安裝 Homebrew [1]#

系統需求#

  • GCC 4.7.0 or newer
  • Linux 2.6.32 or newer
  • Glibc 2.13 or newer
  • 64-bit x86_64 CPU

＃如果你是使用 ARM 和 32-bit x86 的安裝可參考官方文檔

系統為 Debian 系列#

$ sudo apt-get install build-essential curl file git

系統為 Fedora 系列#

$ sudo yum groupinstall 'Development Tools'
$ sudo yum install curl file git
$ sudo yum install libxcrypt-compat # needed by Fedora 30 and up

開始安裝#

$ /bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

設定環境變數#

$ export PATH="/home/linuxbrew/.linuxbrew/bin:$PATH"

或加入 .bashrc (.zshrc)

測試 brew#

$ brew -v

使用方式#

安裝wget

$ brew install wget

常用指令#

以下命令皆在一般 user （非 root）權限下執行，範例中 <package-name> 1 為欲安裝的套件名稱。

套件列表#

#查詢目前已經安裝套件
$ brew list
$ brew list --versions  # 包含各套件的版本號

搜尋套件#

#搜尋目前已經安裝套件(套件過多時使用)
$ brew search <package-name>

套件說明#

#查詢套件的說明資訊
brew info package-name
$ brew info wget

安裝套件#

$ brew install <package-name>

移除套件#

$ brew uninstall <package-name>
$ brew uninstall wget --force  #徹底移除套件的所有檔案

套件更新#

$ brew upgrade <package-name>
$ brew upgrade #若不指定則會更新所有套件，需謹慎使用

套件版本查詢#

$ brew ls <package-name> --versions

更新 Homebrew 及套件清單#

$ brew update

更新 Homebrew 所有套件#

$ brew upgrade

移除指定套件舊版本的安裝檔案#

$ brew cleanup <package-name>

移除所有套件舊版本的安裝檔案#

$ brew cleanup

查詢 Homebrew 手冊#

$ man brew

查詢 brew 指令詳細資訊#

$ brew help info

將套件鎖定版本不更新 / 解除鎖定#

$ brew pin <package-name>
$ brew unpin <package-name>

Homebrew 版本查詢#

$ brew -v

安裝 Hugo#

依照 hugo 官方文件 [3] 依各作業系統不同而安裝。

Homebrew (Linux/OSX)#

$ brew install hugo

Chocolatey (MicroSoft)#

choco install hugo -confirm

或需要“擴充套件”Sass/SCSS版本：

choco install hugo-extended -confirm

建立 Blog 目錄#

使用以下命令創建 Blog 目錄

$ hugo new site <folder_name>

<folder_name> 2 為放置 Blog 目錄之名稱

選擇主題#

建議挑選有持續在 maintain 的主題樣式，能夠從官方主題網站挑選自己喜愛的主題樣式。

$ cd <folder_name>
$ git init
$ git submodule add <https://github.com/????> themes/.

關於 git submodule 介紹與使用方法 [5]

頁面設定#

修改 config.toml 文件，詳細的設定方式可以參照官網[4]

建立第一篇貼文#

hugo new posts/<first-post>.md

透過編輯工具如 vim, visual studio code 等開啟剛剛建立的 <first-post>.md，撰寫想公開的文章並儲存。

markdown 中宣告格式

    ---
    author: "tw1chao"
    title: "如何建立 Hugo 到 Github page 上"
    date: 2022-04-07T13:44:18+08:00
    draft: false
    tags: ["Github page","Hugo"]
    categories: ["Hugo"]
    ShowToc: true
    TocOpen: true
    ---

頁面預覽#

server         高效能網路伺服器 -D,–buildDrafts    載入草稿內容

$ hugo server -D

將 Blog 放到 Github page 上#

在 Blog 目錄底下執行命令建置 public/ 目錄，github page 上只需放置此 hugo 編譯好的網頁檔案，就可將 Blog 公開在網路上。

$ hugo

Github 建立一個 github page 的 Repository，並將其命名為 <username>3.github.io

並透過以下命令 push 至 github repository 中。

$ cd public/
$ git init
$ git add -A
$ git commit -m "Blog 建立及文章發表"
$ git branch -M gh-page
$ git remote add origin https://github.com/<使用者名稱>/<使用者名稱>.github.io.git
$ git push -u origin gh-page

版控 blog 目錄資料#

在 <folder_name> 目錄底下執行

$ git init
$ git add -A
$ git commit -m "Blog 資料備份"
$ git branch -M main
$ git remote add origin https://github.com/<使用者名稱>/<使用者名稱>.github.io.git
$ git push -u origin main
